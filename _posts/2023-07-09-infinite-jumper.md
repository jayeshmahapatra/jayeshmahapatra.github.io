# Infinite Jumper Game for Open Source AI Game Jam

Recently Huggingface organized a game jam called [Open Source AI Game Jam](https://itch.io/jam/open-source-ai-game-jam). The goal was to create an game incorporating Open Source AI tools within 48 hours. I decided to take this oppurtunity to refamiliarize with Unity and explore how new Generative AI tools are being integrated within game dev.


## Game Jam Rules and Theme

The game jam has specific rules to follow:

- The game must be accessible in English.
- It should be playable either on itch.io or Windows.
- Incorporating at least one open source AI tool in the workflow or the game itself is mandatory.
- The game must be completed within 48 hours, adhering to the theme of "Expanding."

To aid participants, Huggingface has provided a comprehensive GitHub page that lists various AI tools suitable for game development: [Compilation of AI tools for Game Dev](https://github.com/simoninithomas/awesome-ai-tools-for-game-dev).

## My Game and Goals

Considering the limited time available to familiarize myself with Unity and game development in general, I decided to choose a game genre that is relatively simple to create. After some consideration, I settled on an **Infinite Jumper** style game. In this genre, players must continuously navigate their character across scrolling platforms by jumping, to remain within the screen's view.

An example of the genre would be [Doodle Jump](https://en.wikipedia.org/wiki/Doodle_Jump). The goal of the game is to keep jumping though the platforms to keep the player from falling.

There are a few reasons why I opted for this genre:

1. Simplicity: Creating the game requires minimal elements, such as a player character, platforms, and some interaction scripts.
2. Time: While the jam allowed 48 hours of dev time, I wanted to spend at best 4-6 hours. The procedural generation aspect of the "Infinite Jumper" genre saved me time that would otherwise be spent designing levels.
2. AI Tool Integration: The beauty of the "Infinite Jumper" concept lies in its flexibility. By reskinning the game with different images generated by generative AI tools, I can transform the player character into a spaceship navigating booster platforms or a wizard exploring an enchanted forest. This allows for easy integration of AI tools while leaving core gameplay logic untouched.

## Open Source AI Tool: Stable Diffusion

For this game jam, I've chosen to utilize the [Stable Diffusion](https://stability.ai/blog/stable-diffusion-public-release). It is a tool that generates images based on textual descriptions. I opted for this tool due to my familiarity with it from previous experiences in generating images.
