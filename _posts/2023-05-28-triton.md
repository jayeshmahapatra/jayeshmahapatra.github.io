## Beyond FastAPI: Using Nvidia Triton for serving ML models


### Serving Models

In today's world, ML engineers are increasingly becoming "full stack" professionals, responsible not only for developing new models but also deploying them. As Python is the language of choice in Machine Learning, it's natural for engineers to gravitate towards Python-based web frameworks for model deployment, with FastAPI being one of the most common choices.

---

### FastAPI

FastAPI is a modern, high-performance web framework designed for building APIs in Python. It offers support for asynchronous request processing and data validation, empowering ML engineers to write REST API endpoints for ML inference.

However, FastAPI wasn't explicitly designed to serve as an inference engine, resulting in a lack of built-in support for ML frameworks and inference-specific features that maximize throughput. While FastAPI is suitable for low loads and scenarios where high throughput isn't necessary, as ML inference scales, it becomes advisable to transition to an inference engine explicitly built for serving ML models.

### Nvidia Triton

Nvidia Triton is an open-source inference serving platform that optimizes and scales machine learning model deployment. Triton outshines FastAPI in several key aspects:

- Framework Support: Triton supports all major ML frameworks, including Tensorflow, PyTorch, TensorRT, Onnx, and more. This built-in framework support simplifies the deployment process for models developed using various ML frameworks.

- Dynamic Batching: Triton has the capability to batch incoming requests in an online fashion, maximizing throughput and reducing latency. This feature allows for efficient handling of multiple inference requests simultaneously.

- Concurrent Model Execution: Triton enables loading multiple models and instances of those models into GPU memory. These models can be executed in parallel, further enhancing throughput and performance.

- GPU Accelerated Preprocessing using DALI Backend: Triton leverages the power of GPUs to accelerate preprocessing tasks using the NVIDIA Data Loading Library (DALI). This results in faster data transformation and enhances the overall inference speed.

- Built-in Integration with Kubernetes and Prometheus for ML-OPs: Triton seamlessly integrates with Kubernetes for efficient container orchestration, enabling scalable deployment and management of ML models. Additionally, its integration with Prometheus facilitates monitoring and observability, essential for ML operations.

For more information about Nvidia triton, you can visit the [official website](https://developer.nvidia.com/nvidia-triton-inference-server).

## Using Triton with Docker: PyTorch Model for Bee vs. Ant Classification

To showcase the deployment workflow of ML models using NVIDIA Triton, FastAPI, and Gradio, we can refer to the [**triton-fastapi-docker**](https://github.com/jayeshmahapatra/triton-fastapi-docker) repository. This repository provides an illustrative example of utilizing Triton with Docker containers, demonstrating a streamlined process for deploying a PyTorch-based machine learning model capable of distinguishing between pictures of bees and ants.

The [**triton-fastapi-docker**](https://github.com/jayeshmahapatra/triton-fastapi-docker) repository consists of the following services:

1. [triton](https://github.com/jayeshmahapatra/triton-fastapi-docker/triton/): Runs the NVIDIA Triton inference engine, serving the PyTorch ML model that distinguishes pictures of bee vs ant.
2. [fastapi](https://github.com/jayeshmahapatra/triton-fastapi-docker/fastapi/): The Python backend communicates with `triton` via gRPC, handling client requests.
3. [gradio](https://github.com/jayeshmahapatra/triton-fastapi-docker/gradio/): Provides a user-friendly interface using Gradio, utilizing both `fastapi` and `triton` for ML inference.

By following the instructions provided in the repository's README, users can easily set up and deploy the PyTorch-based ML model using Docker Compose. The repository also includes a Jupyter notebook located in the `notebooks` directory, guiding users through the process of creating a TorchScript model specifically designed to differentiate between pictures of bees and ants.

This example repository provides insights into how Triton can enhance the deployment process and maximize throughput for ML models. It demonstrates Triton's built-in support for PyTorch and showcases its features such as dynamic batching, concurrent model execution, and GPU-accelerated preprocessing. By exploring this repository, ML engineers can gain practical knowledge and explore the possibilities of efficiently deploying and scaling their own models with Triton.
